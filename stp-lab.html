انتخاب Root Bridge

يُنتخب Root Bridge بناءً على أقل Bridge ID = Priority + MAC Address. حدّد Root Bridge لكل سيناريو.

📐 قاعدة الانتخاب

Bridge ID = Priority (2 بايت) + MAC Address (6 بايت)
• الأولوية الافتراضية: 32768
• يُنتخب السويتش ذو أقل Bridge ID كـ Root Bridge
• إذا تساوت الأولوية → أقل MAC Address يفوز
• الأولوية تأتي بمضاعفات 4096 فقط: 0، 4096، 8192، ... 32768، ...

🏆 سيناريوهات الانتخاب

أدوار منافذ STP

لكل منفذ في السيناريو، حدّد دوره: Root Port، Designated Port، أم Blocked (Alternate).

📋 تعريف أدوار المنافذ

Root Port (RP)
المنفذ على السويتش غير الجذر الذي يصل لـ Root Bridge بأقل تكلفة. كل سويتش غير جذر له Root Port واحد فقط.
Designated Port (DP)
المنفذ المُختار على كل Segment لإرسال البيانات نحو الشبكة. جميع منافذ Root Bridge هي Designated.
Blocked / Alternate
المنفذ المحظور لمنع الحلقات. لا يُرسل ولا يستقبل بيانات عادية، لكن يستقبل BPDUs ليستعد للتفعيل عند الحاجة.
